Tutorials are for doing one thing end to end. The guide explains the mental
model; the reference gives exact behavior. These pages start with a visible
goal and stay close to the target.

Start with [Blink an LED](/tutorials/blink-an-led/) if you are new to the
hardware path. Use [Interactive Workflow](/tutorials/interactive-workflow/)
when you want the edit-loop shape.

The input tutorials split the old button and sensor material into current
surfaces:

- [Read a Button](/tutorials/read-a-button/) for digital input
- [Read a Sensor](/tutorials/read-a-sensor/) for ADC and Machine knobs
- [Read Inputs](/tutorials/read-inputs/) for the compact Machine overview

Use [Build a Calculator](/tutorials/build-a-calculator/) for a local state
exercise, [Build a Small Game](/tutorials/build-a-small-game/) when you have a
Froth Machine nearby, and the Advent of Code pages when you want algorithmic
practice without pretending the language is Python.
