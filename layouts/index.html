What even is this?

Normally, programming a microcontroller goes like this: you write code on your computer, compile it, upload it to the board, and hope. Once it's on there, the board is a sealed box — you can't ask it what it's doing or change your mind without starting the whole cycle over.

Frothy flips that around. The board itself understands the language, so you just talk to it. Type led.on: and the light comes on. Type a new version of a word and it takes effect instantly. Ask the board to show you any part of itself, and it will.

Three things that make Frothy feel different.

{{ partial "icon" "refresh-cw" }}
Live

Change it
while it runs.

Redefine a word and every place that uses it picks up the new version on the spot. No build step, no re-upload, no waiting — the device never stops to listen.

to blink with n [
  repeat n [
    led.on:;  ms: 90    \ on, hold 90ms
    led.off:; ms: 90    \ off, hold 90ms
  ]
]
blink: 3                \ blink three times

…then change 90 to 30 and run it again. Instantly faster.

{{ partial "icon" "eye" }}
Transparent

Nothing is
hidden from you.

Ask the device words and it lists everything it knows. Ask it to show any word and it hands back the source — even on a board that was never yours to begin with.

> words                 \ list what it knows
led.on  led.off  blink

> show @blink           \ read any word's source
to blink with n [ ... ]

The whole device, readable, out in the open.

{{ partial "icon" "pen-line" }}
Readable

If you can read it,
you can write it.

Frothy reads close to plain English. You name a word with to, say what it takes with, and put the steps in brackets. Drastically simpler than C, and friendly to beginners.

to pulse with pin, wait [   \ name it with 'to'
  gpio.high: pin;           \ steps go in brackets
  ms: wait;
  gpio.low: pin
]

No semicolon soup, no toolchain, no datasheet first.
