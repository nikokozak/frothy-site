What even is this?

Normally, programming a microcontroller goes like this: you write code on your computer, compile it, upload it to the board, and hope. Once it's on there, the board is a sealed box — you can't ask it what it's doing.

Frothy flips that around. The board itself understands the language, so you just talk to it. Type led.on: and the light comes on. Type a new version of a word and it takes effect instantly.

{{ partial "mascot" 15 }} The big idea

Three things that make Frothy feel different.

{{ partial "icon" "refresh-cw" }}Live

Change it
while it runs.

Redefine a word and every place that uses it picks up the new version on the spot. No build step, no re-upload — the device never stops to listen.

to blink with n [
  repeat n [
    led.toggle:; ms: 90
  ]
]
blink: 3

…then change 90 to 30 and run again. Instantly faster.

{{ partial "icon" "eye" }}Transparent

Nothing is
hidden from you.

Ask the device words and it lists everything it knows. Ask it to see any one and it hands back the source — even on a board that was never yours.

frothy> words

frothy> see blink
to blink with pin, count, wait [ ... ]

The whole device, readable, out in the open.

{{ partial "icon" "pen-line" }}Readable

Read it, and
you can write it.

Name a word with to, say what it takes with, and put the steps in brackets. Drastically simpler than C, and friendly to beginners.

to pulse with pin, wait [
  gpio.high: pin;
  ms: wait;
  gpio.low: pin
]

No headers, no build flags, no datasheet to read first.

{{ partial "icon" "terminal" }} A look at the language

Seven small programs, read end to end.

word.fr
to led.on [
  gpio.high: $led_builtin
]

led.on:
led.off:
to blink with pin, count, wait [
  repeat count [
    gpio.high: pin;
    ms: wait;
    gpio.low: pin;
    ms: wait
  ]
]

blink: $led_builtin, 5, 200
counter is cells(1)
set counter[0] to 0

repeat 10 [
  set counter[0] to counter[0] + 1
]

counter[0]
to clamp with v, hi [
  if v > hi [ hi ] else [ v ]
]

clamp: 12, 9
clamp: 4, 9
to heartbeat [
  forever [
    led.on:;  ms: 60;
    led.off:; ms: 940
  ]
]

boot is fn [ heartbeat: ]
save
hits is cells(1)

to mark [
  set hits[0] to hits[0] + 1
]

boot is fn [
  on 7 rising [ mark: ]
]
threshold is 25
mode is "idle"

save
clear
restore

notes

A word is a name with code behind it. You write it with to and run it by putting a colon after the name. No parentheses, no main.

$led_builtin is the onboard LED's pin. led.on and led.off already ship in the base image — this just shows what they're made of.

with names the things a word takes. repeat count [ … ] runs the block that many times; ms: waits in milliseconds; ; ends one step.

Arguments go after the colon, separated by commas — here, the pin, how many times, and how long.

is binds a name once and leaves it alone. When you need something that changes, ask for cells(n) — a small run of mutable slots — and write to it with set … to.

Everything that isn't a cell stays put. The last line, counter[0], reads the slot back: 10.

if test [ … ] else [ … ]. The test counts as true when it isn't zero, and the else arm is optional.

A block hands back its last value, so clamp returns the number itself — 9, then 4.

forever [ … ] loops until something stops it — a reset, or Ctrl-C at the prompt.

Bind the boot word and it runs the instant the board powers up. save writes that to flash, so the device wakes up keeping its own time.

on 7 rising [ … ] runs a handler when pin 7 goes high — a button, say.

Frothy only checks between steps — at calls, loop edges, long waits — so a handler never lands in the middle of a write. Cooperative, not preemptive.

Your definitions and values sit in an overlay on top of the base image the board shipped with.

save commits the overlay to flash. clear drops back to base. restore reloads the last save. The state is yours to see, and to take back.

{{ partial "icon" "download" }} Get started

From source to a live board.

There's no package to install yet — you build the frothy CLI from source. It needs Go, and for an ESP32 a one-time toolchain fetch that takes the better part of twenty minutes. After that, flashing and connecting are quick.

  • {{ partial "icon" "check" }} macOS and Linux — Windows isn't supported yet
  • {{ partial "icon" "check" }} ESP32 development boards are the public happy path for now
  • {{ partial "icon" "check" }} Go 1.19+ and a C compiler; the ESP-IDF toolchain installs itself
build from source
$ git clone https://github.com/nikokozak/frothyrewrite
$ cd frothyrewrite
$ make cli            # builds the `frothy` CLI
$ frothy bootstrap    # one-time: fetch the ESP-IDF toolchain
$ frothy flash esp32_devkit_v1 --port /dev/cu.usbserial-0001
$ frothy connect --port /dev/cu.usbserial-0001
